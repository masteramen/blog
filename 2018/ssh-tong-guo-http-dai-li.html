SSH 通过 HTTP 代理

Aug 24, 2018

本文介绍如何通过 HTTPS 代理从受防火墙保护的本地网络连接到 Internet 上的 ssh 服务器。

要求是：

-

您的防火墙必须允许通过代理进行 HTTPS 连接

-

您需要具有 ssh 正在侦听的服务器的 root 访问权限

[配置 ssh 服务器]（配置 ssh 服务器）

ssh 守护进程需要在 443 端口上侦听。要完成此操作，只需编辑此文件（在 debian 系统上）/etc/ssh/sshd_config并添加以下行：

Port 443

然后重启守护进程：

sudo /etc/init.d/ssh restart

[配置客户端]（配置客户端）

我想你是在 Linux 系统上（例如 debian）。首先，您必须编译[connect binary]（http://www.meadowy.org/~gotoh/projects/connect），它将帮助您的ssh客户端使用代理（在我们的例子中为HTTPS）。然后，您必须配置您的ssh客户端，以告诉他在尝试连接到您的ssh服务器时使用HTTPS代理。

1.

-

在 debian 系统上，只需安装connect-proxy包： sudo apt-get install connect-proxy

-

在其他 Linux 系统上，您必须编译它： cd /tmp/ wget http://www.meadowy.org/~gotoh/ssh/connect.c gcc connect.c -o connect sudo cp connect /usr/local/bin/ ; chmod +x /usr/local/bin/connect

2.

Configure your ssh client. Open or create your ~/.ssh/config file and add these lines : 配置您的 ssh 客户端。打开或创建〜/.ssh/config文件并添加以下行：(zh_CN) ## Outside of the firewall, with HTTPS proxy Host my-ssh-server-host.net ProxyCommand connect -H proxy.free.fr:3128 %h 443 ## Inside the firewall (do not use proxy) Host * ProxyCommand connect %h %p

3.

然后祈祷并测试连接： ssh my-ssh-server-host.net

[通过隧道连接到另一台服务器]（通过隧道连接到另一台服务器）

例如，连接到 sshgithub.com：

Host github.com
  ProxyCommand=ssh my-ssh-server-host.net "/bin/nc -w1 %h %p"
